## Template Customization

### Customize for Your Team

Edit `team-settings.json` to match your team's needs:

#### 1. Model Selection

```json
{
  "ai": {
    "defaultModel": "sonnet",
    "agentModelSelection": {
      "planning": "sonnet",      // Use Sonnet for complex planning
      "coding": "sonnet",         // Use Sonnet for implementation
      "review": "sonnet",         // Use Sonnet for thorough reviews
      "testing": "haiku",         // Use Haiku for faster testing
      "documentation": "haiku"    // Use Haiku for documentation
    }
  }
}
```

**Model Recommendations**:
- **Sonnet**: Complex tasks, critical decisions, thorough analysis
- **Haiku**: Fast tasks, simple operations, documentation

#### 2. Quality Gates

```json
{
  "workflow": {
    "qualityGates": {
      "requirePlanApproval": true,      // Must approve plans before coding
      "requireCodeReview": true,        // Must pass code review
      "requireDocReview": true,         // Must pass doc review
      "requireTestReview": true         // Must pass test review
    }
  }
}
```

**Recommended Settings**:
- **Strict Teams**: All `true` (highest quality)
- **Agile Teams**: `requirePlanApproval: true`, others `false`
- **Solo Developers**: All `false` (maximum speed)

#### 3. Git Worktrees

```json
{
  "workflow": {
    "git": {
      "useWorktrees": true,              // Enable worktree isolation
      "worktreePath": ".worktree",       // Path for worktrees
      "autoCleanup": true,               // Auto-cleanup after merge
      "cleanupStages": 4                 // 4-stage cleanup process
    }
  }
}
```

**When to Use Worktrees**:
- âœ… Multiple features in parallel
- âœ… Team development
- âœ… Complex feature branches
- âšª Simple, single-feature work

#### 4. Testing Configuration

```json
{
  "workflow": {
    "testing": {
      "autoRunTests": false,             // Don't auto-run (ask first)
      "requireTestsToPass": true,        // Block merge if tests fail
      "testTimeout": 300000              // 5 minutes timeout
    }
  }
}
```

**Testing Strategies**:
- **CI/CD**: `autoRunTests: true`, `requireTestsToPass: true`
- **Manual**: `autoRunTests: false`, `requireTestsToPass: true`
- **Development**: `autoRunTests: false`, `requireTestsToPass: false`

#### 5. Documentation

```json
{
  "workflow": {
    "documentation": {
      "autoGenerateDocs": true,          // Generate docs automatically
      "requireDocumentation": true,      // Require docs for completion
      "docFormat": "markdown"            // Use Markdown format
    }
  }
}
```

## Usage Examples

### Example 1: Setup for New Project

```bash
# 1. Create new repository
git init my-project
cd my-project

# 2. Copy team template
mkdir -p .claude
cp /path/to/lilylab-marketplace/templates/team-settings.json .claude/settings.json

# 3. Customize for your team (edit .claude/settings.json)

# 4. Commit configuration
git add .claude/settings.json
git commit -m "Add LilyLab Claude Code configuration"

# 5. Push to remote
git push origin main
```

### Example 2: Migrate Existing Project

```bash
# 1. Navigate to project
cd my-existing-project

# 2. Create .claude directory
mkdir -p .claude

# 3. Copy minimal template (start simple)
cp /path/to/lilylab-marketplace/templates/minimal-settings.json .claude/settings.json

# 4. Test with team
git add .claude/settings.json
git commit -m "Add Claude Code plugins"
git push origin main

# 5. Upgrade to full template later if needed
cp /path/to/lilylab-marketplace/templates/team-settings.json .claude/settings.json
```

### Example 3: Personal Project

```bash
# 1. Create personal settings (not committed)
mkdir -p .claude
cp /path/to/lilylab-marketplace/templates/minimal-settings.json .claude/settings.json

# 2. Add to .gitignore
echo ".claude/settings.json" >> .gitignore

# 3. Customize freely without affecting repository
```

## Troubleshooting

### Configuration Not Loading

1. **Check file location**: Must be `.claude/settings.json`
2. **Check JSON syntax**: Use validator above
3. **Trust repository**: Claude Code must trust the folder
4. **Restart Claude Code**: Exit and restart

### Plugins Not Installing

1. **Check marketplace source**: Verify GitHub URL
2. **Check network**: Ensure internet access
3. **Check plugin name**: Must match exactly
4. **Manual installation**: Try `/plugin install` manually
